# AMC Report Doctrine

## Core Report Principle

The AMC report is a structural decision brief. It does not tell the client what to do. It shows the structure of the decision they are facing.

## Report Structure to Preserve

### Executive Dashboard
Purpose: one-page structured overview of the decision.

Visible labels:
- Structural Reading
- Core Structural Insight
- Risk Summary
- Value Summary
- Mobility Summary
- Temperament Summary

Do not use: Decision Verdict.

### Executive Overview
Purpose: short narrative overview of the case and the main structural tension.

### External Snapshot
Purpose: external context pressure for a single-path or single-decision case.

Format:
- Section title
- Optional subtitle/title line
- Market Direction
- Competition Pressure
- Economic Pressure
- Transition Friction

This is a narrative overlay, not a score.

### External Comparative Snapshot
Purpose: comparison between Option A and Option B.

This section should prioritize instant comparison.

Required order:
- Section title
- Comparison table first
- Compact legend second
- Comparative Reading
- Comparative Implication

Table columns:
- Dimension
- Option A
- Option B

Rows:
- Market Direction
- Competition Pressure
- Economic Pressure
- Transition Friction

Status system:

Market Direction:
- ▲ Supportive
- ◆ Mixed
- ▼ Constrained

Competition / Economic / Transition:
- ○ Contained
- ◐ Moderate
- ● Elevated

Comparative Reading and Comparative Implication must remain short. The table is the primary device.

### Structural Risk Diagnosis
Purpose: internal structural exposure analysis.

Includes:
- Internal Structural Snapshot
- D1–D5 diagnosis
- FIFWM
- Safety Margin
- Implications

### Career Value Structure
Format:
- Current Path
- Transition Path
- Value Reading
- Value Implication

### Career Mobility Structure
Format:
- Mobility Type
- Mobility Reading
- Mobility Implication

Only the selected mobility type is shown in the report.

### Strategic Temperament
Format:
- Temperament Profile
- Temperament Reading
- Temperament Implication

### Structural Scorecard / Five-Factor Matrix
Purpose: compact scored synthesis layer.

Keep 3-slot bars and restrained visuals.

### Hybrid Exploration Plan
Format:
- Experiment 1
- Experiment 2
- Experiment 3
- 90-day execution map

### Decision Conditions
Purpose: show under what structural conditions commitment becomes more supportable.

Visible labels:
- Decision Conditions
- Commitment Condition

Do not use:
- Decision Gate
- Go / No-Go wording
- recommendation-style labels

## Scored vs Non-Scored Layers

Scored layers:
- FIFWM
- Safety Margin
- Structural Scorecard / Five-Factor Matrix

Non-scored layers:
- Executive Dashboard
- Executive Overview
- External Snapshot
- External Comparative Snapshot
- Internal Structural Snapshot
- Career Value Structure
- Career Mobility Structure
- Strategic Temperament
- Hybrid Exploration Plan
- Decision Conditions

Do not visually blur these categories.

## Single-Case vs Comparative-Case Logic

Single-case report should primarily show External Snapshot.

Comparative-case report should primarily show External Comparative Snapshot.

Do not make both primary sections visually compete on the same page.

Comparative mode should feel intentionally different and comparison-led.

## Visual Direction

Use:
- premium
- calm
- elegant
- dense but readable
- consulting-style
- private wealth note / strategy memo feel
- clean hierarchy
- restrained contrast
- minimal visual clutter

Avoid:
- startup SaaS dashboard style
- flashy KPI cards
- neon colors
- gamified UI
- cartoon icons
- motivational/self-help aesthetics
- overly decorative design
