# AMC Operating System

## Project Identity

**AMC — All of My Career** is a structural lens and reasoning tool for serious career decisions.

AMC does not decide for the client. AMC does not recommend. AMC structures the decision.

The product should feel like a private strategic note or consulting-grade decision brief, not a coaching product, motivational product, or startup SaaS dashboard.

## Working Roles

### Human Owner / Approval Layer
- Sets strategic direction
- Reviews intermediate outputs
- Approves brand, business, and product decisions
- Decides what to keep, revise, or discard

### ChatGPT / AI Project Lead
- Maintains AMC logic and brand consistency
- Converts strategy into executable tasks
- Reviews Codex outputs against AMC doctrine
- Produces next-step task prompts
- Protects terminology, hierarchy, and tone

### Codex / Execution Layer
- Implements files and code
- Produces prototypes
- Refactors and tests
- Opens changes for review where possible

## Operating Loop

1. Human approves the next product direction.
2. ChatGPT writes or updates a Codex task file.
3. Codex executes the task in the repository.
4. ChatGPT reviews the output for AMC consistency.
5. Human approves or redirects.
6. Repeat.

## Current Productization Stage

AMC is past initial ideation. The current stage is **integration, documentation, and development operations**.

The immediate objective is not to invent new AMC logic. The objective is to consolidate existing assets into a durable product system:

- AMC master doctrine
- Report architecture
- Brand doctrine
- Terminology rules
- Web/report prototype
- Codex task pipeline
- Review and approval loop

## Non-Negotiable Philosophy

- Do not recommend.
- Do not decide.
- Do not use coaching language.
- Do not use motivational language.
- Do not use Go / No-Go framing.
- Show decision structure, risk concentration, trade-offs, and commitment conditions.

## Preferred Language

Use language such as:

- appears
- remains
- suggests
- indicates
- reflects
- preserves
- increases
- reduces
- is driven by
- is constrained by

Avoid language such as:

- you should
- best choice
- right answer
- strong recommendation
- decision verdict
- go / no-go

## Primary Automation Goal

Move from manual prompt copy/paste into a repository-centered workflow where task files, issues, branches, and pull requests become the operating interface.

The desired system is:

```text
Human approval
  ↓
ChatGPT task design / review
  ↓
Codex implementation
  ↓
GitHub PR or commit
  ↓
ChatGPT consistency review
  ↓
Human approval
```
